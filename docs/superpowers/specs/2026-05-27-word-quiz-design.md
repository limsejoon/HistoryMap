# 단어 퀴즈 웹페이지 디자인 스펙

## 개요

한→영 플래시카드 방식의 단어 퀴즈. 단일 HTML 파일로 동작하며 별도 서버·빌드 불필요.

---

## 핵심 결정사항

| 항목 | 결정 |
|------|------|
| 퀴즈 형식 | 플래시카드 (앞면 탭 → 뒤집기) |
| 언어 방향 | 한국어 → 영어 |
| 애니메이션 | 3D 플립 (CSS perspective, Y축 180° 회전) |
| 색상 테마 | 웜 오렌지 (배경 #fef9f0, 포인트 #f97316) |
| 세션 구성 | 랜덤 셔플 (매 세션 순서 무작위) |
| 단어 수 | 5개 하드코딩 |
| 구현 방식 | 단일 HTML/CSS/JS 파일 |

---

## 화면 흐름

```
[앞면: 한국어] → 탭 → [3D 플립] → [뒷면: 영어 정답]
                                         ↓
                                  [알았어 / 몰랐어]
                                         ↓
                              다음 카드 or [완료 화면]
```

### ① 앞면 (카드 미뒤집힘 상태)
- 상단: 진행 표시 (N / 5) + 프로그레스 바
- 중앙: 오렌지 그라디언트 카드에 한국어 단어
- 카드 하단: "탭하여 뒤집기" 안내 텍스트
- 하단: 현재 알았어/몰랐어 누적 카운트

### ② 뒷면 (카드 뒤집힌 상태)
- 카드 상단: 작게 한국어 단어 (컨텍스트용)
- 카드 중앙: 영어 정답 (크게)
- 하단 버튼 2개: `✗ 몰랐어` (빨강) / `✓ 알았어` (초록)
- 버튼 선택 즉시 다음 카드로 이동

### ③ 완료 화면
- 이모지 + "퀴즈 완료!" 타이틀
- 점수: X / 5 (오렌지 강조)
- 알았어 / 몰랐어 분리 카운트
- `다시 하기 (셔플)` 버튼 → 순서 다시 섞어 재시작

## 데이터 구조

```js
const words = [
  { front: "사과", back: "apple" },
  { front: "책",  back: "book" },
  { front: "물",  back: "water" },
  { front: "친구", back: "friend" },
  { front: "집",  back: "house" },
];
```

단어 추가/수정은 이 배열만 편집하면 됨.

---

## 상태 관리

```
cards[]      — 셔플된 단어 배열 (세션 시작 시 생성)
current      — 현재 카드 인덱스 (0~4)
flipped      — 현재 카드 뒤집힘 여부 (boolean)
score        — { know: 0, dontKnow: 0 }
```

---

## 애니메이션 구현

- 카드 컨테이너에 `perspective: 1000px`
- 앞면/뒷면을 `position: absolute`로 겹침, 뒷면은 `rotateY(180deg)` 초기값
- `backface-visibility: hidden`으로 뒷면 숨김
- 뒤집기 시 컨테이너에 `rotateY(180deg)` 적용, transition 0.5s ease
