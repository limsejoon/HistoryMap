# 유럽 중세 역사 게임 MVP — 디자인 문서

**날짜:** 2026-05-28  
**프로젝트:** HistoryMap  
**상태:** 승인됨

---

## 1. 개요

기존 `index.html` (Leaflet + OSM 지도)을 유럽 중세 역사 퀴즈 게임으로 발전시킨다.  
서버·DB·로그인 없이 정적 파일만으로 동작하며, GitHub Pages / Vercel 등 정적 호스팅에서 바로 배포 가능하다.

---

## 2. 파일 구조

```
HistoryMap/
├── index.html      # 기존 파일 확장 (지도 + UI 전체)
└── events.json     # 도시·사건·퀴즈 데이터 (신규)
```

추가 라이브러리 없음. Leaflet + OSM 그대로 유지.

---

## 3. UI 레이아웃

```
┌──────────────────────────────┬──────────────────┐
│                              │  🏛 콘스탄티노플  │
│   Leaflet 지도               │  Byzantine …      │
│   (유럽 중심, zoom 4)        ├──────────────────┤
│                              │  📜 주요 사건     │
│  ⭐ Score: 2  (좌상단)       │  330  …           │
│                              │  1054 …           │
│  ● 마커 4개 (금색)           │  1453 …           │
│  ● 활성 마커 (빨간색)        ├──────────────────┤
│                              │  ❓ 퀴즈          │
│                              │  [보기 4개 버튼]  │
│                              │  ✓ 정답! +1점     │
└──────────────────────────────┴──────────────────┘
```

- **지도:** 유럽 중심 (`setView([48, 18], 4)`)
- **점수 배지:** 좌상단 고정, `position: fixed`
- **사이드 패널:** 우측 고정 (300px), 항상 표시. 초기 상태엔 "← 도시를 클릭하세요" 안내 문구
- **퀴즈:** 도시당 1회. 정답 선택 후 버튼 비활성화 (재시도 불가)
- **닫기 버튼 없음:** 다른 마커 클릭 시 패널 내용 교체 (MVP 범위)

---

## 4. 상호작용 흐름

1. 페이지 로드 → `fetch('events.json')` → 도시 데이터 로딩
2. 각 도시 좌표에 Leaflet 마커 자동 생성
3. 마커 클릭 → 사이드 패널 열림, 해당 도시 사건·퀴즈 표시
4. 퀴즈 보기 클릭 → 정답이면 초록색 + score+1, 오답이면 빨간색
5. 퀴즈 완료 후 버튼 전체 비활성화 (같은 도시 재방문 시 결과 유지)
6. 다른 마커 클릭 → 패널 내용 교체

## 6. 점수 시스템

- 전역 변수 `score` (정수, 초기값 0)
- 퀴즈 정답 시 +1, 화면 좌상단 배지 즉시 업데이트
- 페이지 새로고침 시 초기화 (세션 유지 없음 — MVP 범위)
- 도시당 1회만 채점 (재방문 시 이미 답한 상태 유지)

---

## 7. 기술 제약

- Leaflet 1.9.4 + OpenStreetMap (기존 그대로)
- 추가 npm 패키지 없음
- `fetch('events.json')` 사용 → `file://` 직접 열기 불가, 로컬 서버 필요
  - 로컬 테스트: `npx serve .` 또는 VS Code Live Server
  - 배포: GitHub Pages / Vercel (정적 호스팅이면 모두 동작)
- IE 지원 없음 (ES6+ fetch, const/let 사용)
